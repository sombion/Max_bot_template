# Max Bot Template

Шаблон для быстрого старта разработки бота на платформе Max с использованием `maxapi`.

## Что есть в шаблоне

- базовая структура проекта на Python
- загрузка настроек из `.env`
- запуск бота
- обработчик старта `/start`
- пример текстового обработчика
- пример inline-кнопок и callback-обработчиков

## Требования

- Python `3.12+`
- токен бота в Max

## Подготовка

1. Перейдите в папку проекта.
2. Создайте файл `.env` в корне проекта.
3. Добавьте в него токен бота:

```env
BOT_TOKEN=ваш_токен_бота
```

Без файла `.env` и переменной `BOT_TOKEN` приложение не запустится.

## Установка зависимостей

### Вариант 1. Через `uv`

Рекомендуемый вариант:

```bash
uv sync
```

### Вариант 2. Через `pip`

```bash
python -m venv .venv
source .venv/bin/activate
pip install -r requirements.txt
```

## Запуск бота

Из корня проекта:

### Если используете `uv`

```bash
uv run python -m app.main
```

### Если используете виртуальное окружение и `pip`

```bash
python -m app.main
```

После запуска бот начнет работать в режиме polling.

## Структура проекта

```text
app/
  config.py            # настройки и загрузка переменных окружения
  main.py              # точка входа
  handlers/
    start.py           # обработчик запуска бота и команды /start
    main.py            # обработчики сообщений и callback-кнопок
  keyboards/
    main.py            # генерация inline-клавиатуры
```

## Где менять логику бота

- Основной запуск находится в `app/main.py`
- Переменные окружения описаны в `app/config.py`
- Команду `/start` и приветственное сообщение можно менять в `app/handlers/start.py`
- Ответы на пользовательские сообщения находятся в `app/handlers/main.py`
- Кнопки и payload для callback находятся в `app/keyboards/main.py`

## Пример сценария разработки

1. Создайте и заполните `.env`
2. Установите зависимости
3. Запустите бота
4. Измените обработчики в `app/handlers/`
5. Перезапустите приложение после изменений

## Полезно знать

- Логирование выполняется через `loguru`
- Настройки читаются через `pydantic-settings`
- Если бот не запускается, сначала проверьте наличие `.env` и правильность `BOT_TOKEN`
